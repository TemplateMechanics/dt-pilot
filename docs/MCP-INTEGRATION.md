# MCP Integration

dt-pilot integrates the official **Dynatrace MCP server** (`@dynatrace-oss/dynatrace-mcp-server`) so AI agents have first-party access to DQL, problems, vulnerabilities, entities, and Davis Copilot. This document describes how the integration is wired, how to configure auth, and how to toggle servers safely.

## Files involved

| File | Role |
|---|---|
| [`.vscode/mcp.json`](../.vscode/mcp.json) | Shareable workspace MCP template. Committed. References env vars by name — never holds secrets. |
| [`.vscode/mcp.session.json`](../.vscode/mcp.session.json) | Per-developer override. **Gitignored.** Generated by `New-McpSessionConfig.ps1` from the template + catalog. |
| [`.vscode/mcp.servers.catalog.json`](../.vscode/mcp.servers.catalog.json) | Authoritative catalog of MCP servers dt-pilot knows how to launch, with default state and required env vars. |
| [`.vscode/schemas/mcp-servers-catalog.schema.json`](../.vscode/schemas/mcp-servers-catalog.schema.json) | JSON Schema validating the catalog. |
| [`scripts/Start-DynatraceMcpServer.ps1`](../scripts/Start-DynatraceMcpServer.ps1) | Stdio launcher invoked by `.vscode/mcp.json`. |
| [`scripts/Test-DynatraceMcpReadiness.ps1`](../scripts/Test-DynatraceMcpReadiness.ps1) | Pre-flight check (Node version, env vars, catalog parse). |
| [`scripts/Set-McpServerState.ps1`](../scripts/Set-McpServerState.ps1) | Toggle a server's `disabled` flag in `mcp.json`. Respects `alwaysEnabled`. |
| [`scripts/New-McpSessionConfig.ps1`](../scripts/New-McpSessionConfig.ps1) | Generate a per-developer `mcp.session.json` from the template + catalog. |
| [`scripts/Test-McpConfigSecrets.ps1`](../scripts/Test-McpConfigSecrets.ps1) | Scan committed (or staged) MCP configs for hardcoded tokens / live tenant URLs / bearer-in-URL. |

## Architecture

```text
VS Code / Claude Code / Cursor / Codex
        │  (MCP stdio framing)
        ▼
.vscode/mcp.json   ──┐
                     │  (per-developer overrides, if the client supports them)

                     ▼

                     │  (stdio MCP transport)
                     ▼
Dynatrace platform  (DQL, problems, entities, vulnerabilities, Davis Copilot)
```

## Authentication

The Dynatrace MCP server supports three auth modes; dt-pilot configures whichever the user provides via environment variables:

| Mode | Env vars | When to use |
|---|---|---|
| Platform token | `DT_ENVIRONMENT`, `DT_PLATFORM_TOKEN` | Local development, scratch environments, fast local DQL |
| OAuth client credentials | `DT_ENVIRONMENT`, `OAUTH_CLIENT_ID`, `OAUTH_CLIENT_SECRET` | CI, production deploys, account-management operations |
| Browser SSO | `DT_ENVIRONMENT` only | Interactive use only; the server opens a browser window |

See [`AUTHENTICATION.md`](AUTHENTICATION.md) for the full setup walkthrough (token creation, OAuth client provisioning, scope requirements).

## How to add a new MCP server

1. Add the server entry to [`.vscode/mcp.servers.catalog.json`](../.vscode/mcp.servers.catalog.json):
   - `id` (kebab-case, must match the key under `servers` in `mcp.json`)
   - `displayName`, `description`, `homepage`
   - `defaultEnabled`, `alwaysEnabled`
   - `requiresEnv`, `acceptsOneOfEnv`
2. Add the corresponding server entry to [`.vscode/mcp.json`](../.vscode/mcp.json) (command, args, env).
3. If the server prompts the user for any input, add a matching entry to the `inputs` array in `mcp.json`.
4. Open a PR. The catalog schema check (PR&nbsp;6 CI) and `Test-McpConfigSecrets.ps1` will validate the addition.

## How to toggle a server

```powershell
./scripts/Set-McpServerState.ps1 -Server context7 -Enable
./scripts/Set-McpServerState.ps1 -Server context7 -Disable
```

`Set-McpServerState.ps1` refuses to disable a server marked `alwaysEnabled` in the catalog. Never hand-edit `.vscode/mcp.json` to flip a server on/off — the script preserves JSON formatting and centralises the validation.

## Secret hygiene

**Never commit a secret to `.vscode/mcp.json`.** The committed file references env vars by name. Per-developer secrets live in `.vscode/mcp.session.json` (gitignored) or in environment variables that the launcher script reads.

The pre-commit gate (`Pre-Commit.ps1`, PR&nbsp;6) runs `Test-McpConfigSecrets.ps1 -StagedOnly` and blocks any commit that includes:

- A Dynatrace token literal (any `dt0XX.` prefix)
- A live tenant URL (`*.live.dynatrace.com`, `*.apps.dynatrace.com`, `*.dynatracelabs.com`)
- A credential embedded in a URL (`https://user:token@...`)

If the gate fires, move the secret to `.vscode/mcp.session.json` or to an environment variable, then re-stage.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| MCP server fails to start | `npx` not on PATH; Node.js < 20 | Install Node.js 20+; run `./scripts/Test-DynatraceMcpReadiness.ps1` |
| MCP server connects but every tool call fails with 401 | Auth env vars not set or wrong | Re-check `DT_PLATFORM_TOKEN` / OAuth; verify scopes (see `AUTHENTICATION.md`) |
| MCP client says "JSON parse error" on every response | The launcher script accidentally wrote to stdout | Diagnostics in `Start-DynatraceMcpServer.ps1` must go to stderr — never to stdout |
| `Set-McpServerState.ps1` refuses to disable `dynatrace` | The catalog marks it `alwaysEnabled` | Intentional. Edit the catalog if you genuinely need it off (uncommon). |

## Out of scope

- **OAuth client provisioning** is out of band — provision via the Dynatrace UI / Account Management API and store the credentials in env vars / a secrets manager.
- **Grail query budget management** lives at the tenant level. Set `DT_GRAIL_QUERY_BUDGET_GB` if you need to lower the default 1000 GB cap.
